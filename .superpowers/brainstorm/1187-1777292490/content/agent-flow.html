Agent 调度流程

用户创建帖子到自动发布，全链路

👤 用户 → 上传图片 + 填写文案 + 设时间
│
🌐 FastAPI → POST /posts/create → 写入 SQLite
│
🤖 Agent → 检测新帖子 → 组装内容 → 注册定时任务
│
⏰ APScheduler → 到时间触发 → 调用发布接口
│
📤 Publisher → 执行发布 → 写日志 → 更新帖子状态
│
✓ 发布完成 → posts.status = published

Agent 具体做了什么

Agent 不是一个独立的"思考者"，而是内容 + 调度的编排器，具体职责：

步骤 动作 输入 → 输出
1. 接收 用户创建帖子 图片路径 + 标题 + 正文 + 时间 → 组装为 Post 对象
2. 调度 注册定时任务 Post → APScheduler.add_job()
3. 触发 到时间执行 取出 Post → 调用 Publisher
4. 记录 写日志 + 更新状态 结果 → publish_logs + posts.status

关键点

  • Agent 不写内容，只编排流程
  • APScheduler 负责定时触发，Agent 负责调度决策
  • Publisher 是独立接口，换发布方式不动 Agent
  • 服务重启后，APScheduler 从数据库恢复待执行任务
